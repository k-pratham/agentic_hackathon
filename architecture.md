# Architecture Specification: Secure Section-Wise Document Ingestion Pipeline

## 1. Executive Summary & System Objectives

This document details the architecture for an end-to-end regulatory document processing pipeline. The system eliminates manual, high-latency execution paths by processing documents entirely in-memory or via streamlined external API microservices.

### Core Requirements:

* **High-Fidelity Document Layout Extraction:** Utilize visual language models to convert document pages directly into structured, semantic Markdown format, preserving tables and section hierarchies cleanly.
* **Context-Preserving Vector Search:** Implement section-wise chunking mapped to long-context embedding models to eliminate truncation losses.
* **Multi-Tenant Role Isolation:** Secure all processed data at the storage layer, ensuring both exact relational parameters and semantic vectors are strictly segregated by user department boundaries using Oracle VPD and Elasticsearch term filtering.
* **Asynchronous Processing Stability:** Design an architecture that functions reliably on resource-constrained hardware in development (8GB RAM, 4 CPU Cores) while scaling horizontally to support enterprise production levels (70,000+ documents).

CREATE OR REPLACE PACKAGE BODY application_security_ctx AS
    g_dept_id NUMBER;

    PROCEDURE set_session_user(p_email IN VARCHAR2) IS
    BEGIN
        SELECT dept_id INTO g_dept_id 
        FROM person_master 
        WHERE person_email = p_email AND is_active = 1;
    EXCEPTION
        WHEN NO_DATA_FOUND THEN
            g_dept_id := -1; 
    END set_session_user;

    FUNCTION get_user_dept_predicate(p_schema IN VARCHAR2, p_table IN VARCHAR2) RETURN VARCHAR2 IS
    BEGIN
        IF g_dept_id IS NULL THEN
            RETURN '1=1'; -- Handle administrative bypass if necessary
        ELSE
            RETURN 'dept_id = ' || g_dept_id;
        END IF;
    END get_user_dept_predicate;
END application_security_ctx;

## 5. Python Pipeline Worker Implementation

This core orchestration script processes documents through `PyMuPDF` and the `ONNX` runtime environment while routing API transactions to external instances.

```python
import os
import re
import base64
import httpx
import fitz  # PyMuPDF
import numpy as np
from transformers import AutoTokenizer
from optimum.onnxruntime import ORTModelForFeatureExtraction

class IngestionPipelineWorker:
    def __init__(self, numarkdown_url: str, elastic_url: str):
        self.numarkdown_url = numarkdown_url
        self.elastic_url = elastic_url
        
        # Initialize Nomic Long-Context Model via CPU-Optimized ONNX Engine
        self.model_id = "nomic-ai/nomic-embed-text-v1.5"
        self.tokenizer = AutoTokenizer.from_pretrained(self.model_id)
        self.embedding_model = ORTModelForFeatureExtraction.from_pretrained(
            self.model_id, 
            export=True,
            provider="CPUExecutionProvider"
        )
        self.network_client = httpx.Client(timeout=httpx.Timeout(180.0, connect=10.0))

    def extract_pdf_pages_to_images(self, pdf_path: str) -> list:
        base64_images = []
        doc = fitz.open(pdf_path)
        for page_num in range(len(doc)):
            page = doc.load_page(page_num)
            pixmap = page.get_pixmap(matrix=fitz.Matrix(2.0, 2.0), alpha=False)
            base64_images.append(base64.b64encode(pixmap.tobytes("png")).decode("utf-8"))
        doc.close()
        return base64_images

    def convert_image_to_markdown_via_api(self, base64_image: str) -> str:
        payload = {
            "model": "numind/NuMarkdown-8B-Thinking",
            "image": f"data:image/png;base64,{base64_image}",
            "prompt": "Convert this document layout into standardized GitHub-Flavored Markdown text transparently."
        }
        response = self.network_client.post(f"{self.numarkdown_url}/v1/completions", json=payload)
        response.raise_for_status()
        raw_output = response.json().get("text", "")
        return re.sub(r"<think>.*?</think>", "", raw_output, flags=re.DOTALL).strip()

    def split_document_into_explicit_sections(self, full_text: str) -> list:
        clean_text = re.sub(r"(?i)page\s+\d+\s+of\s+\d+", "", full_text)
        chunks = re.split(r"(?=\n(?:#{1,6}\s+|\d+\.\d+\s+[A-Z]))", clean_text)
        return [chunk.strip() for chunk in chunks if chunk.strip()]

    def generate_matryoshka_embedding(self, text_chunk: str) -> list:
        enriched_text = f"search_document: {text_chunk}"
        inputs = self.tokenizer(enriched_text, padding=True, truncation=True, max_length=8192, return_tensors="pt")
        outputs = self.embedding_model(**inputs)
        raw_vectors = outputs.last_hidden_state.mean(dim=1).detach().numpy()[0]
        
        truncated_vector = raw_vectors[:384]
        normalized_vector = truncated_vector / np.linalg.norm(truncated_vector)
        return normalized_vector.tolist()

    def process_file_pipeline(self, pdf_path: str, meta: dict):
        images = self.extract_pdf_pages_to_images(pdf_path)
        
        aggregated_markdown = ""
        for idx, img in enumerate(images):
            page_md = self.convert_image_to_markdown_via_api(img)
            aggregated_markdown += f"\n\n\n" + page_md
            
        document_sections = self.split_document_into_explicit_sections(aggregated_markdown)
        
        for index, section_content in enumerate(document_sections):
            vector = self.generate_matryoshka_embedding(section_content)
            es_payload = {
                "doc_id": meta["doc_id"],
                "insp_id": meta["insp_id"],
                "dept_id": meta["dept_id"],
                "document_type": meta["document_type"],
                "entity_name": meta["entity_name"],
                "report_year": meta["report_year"],
                "section_index": index,
                "text_content": section_content,
                "content_vector": vector
            }
            self.network_client.post(f"{self.elastic_url}/regulatory_documents_index/_doc", json=es_payload)

    def close(self):
        self.network_client.close()

```

## 6. Secure Query Execution (Multi-Tenant Segregation Integration)

When an authenticated user issues a request, the system establishes their context in Oracle, retrieves their department identifier, and propagates that security boundary directly into Elasticsearch.

### Production Hybrid Security Search Invocation Structure:

```python
import oracledb

def secure_search_coordinator(user_email: str, query_text: str, user_vector: list, elastic_url: str, oracle_conn):
    # Step 1: Initialize Oracle Security Context
    with oracle_conn.cursor() as cursor:
        # Securely set session context via PL/SQL package
        cursor.execute("BEGIN application_security_ctx.set_session_user(:email); END;", [user_email])
        
        # Verify the context was successfully mapped to a valid department
        cursor.execute("SELECT dept_id FROM person_master WHERE person_email = :email AND is_active = 1", [user_email])
        user_dept_row = cursor.fetchone()
        
        if not user_dept_row:
            raise PermissionError("Access Denied: Invalid or inactive user identity signature.")
        authorized_dept_id = user_dept_row[0]

    # Step 2: Formulate the hybrid search payload with mandatory term pre-filter
    search_payload = {
        "query": {
            "bool": {
                "must": [ { "match": { "text_content": query_text } } ],
                "filter": [ { "term": { "dept_id": authorized_dept_id } } ]
            }
        },
        "knn": {
            "field": "content_vector",
            "query_vector": user_vector,
            "k": 5,
            "num_candidates": 50,
            "filter": {
                "term": { "dept_id": authorized_dept_id }
            }
        }
    }
    
    # Step 3: Dispatch secured query to Elasticsearch
    with httpx.Client() as client:
        response = client.post(f"{elastic_url}/regulatory_documents_index/_search", json=search_payload)
        return response.json()

```

## 7. Scale Optimization Matrix (Dev vs. Production Profile)

Because the processing pipeline is entirely stateless, scaling up from development parameters to production levels requires no code modifications.

| Operational Component | Development Profile (8GB Machine) | Production Profile (Target Cluster Scale) |
| --- | --- | --- |
| **Document Volume Target** | ~100 Testing Reports | 70,000+ Production Scale Documents |
| **Pipeline Core Concurrency** | Max 2–3 active parallel worker loops | 24+ parallel worker threads via Celery/Redis |
| **Oracle Location** | External Oracle Express/Standard Node | Enterprise RAC or Active Data Guard with UCP |
| **Elasticsearch Node Target** | External Single Node Sandbox Index | High-availability multi-node cluster with sharding |
| **Vector Index Performance** | `int8_hnsw` Quantization | `int8_hnsw` or standard `hnsw` on dedicated RAM nodes |
| **NuMarkdown Hosting Node** | Dedicated Remote Machine GPU Engine | Auto-scaling vLLM / Ollama Cluster Pool on GPUs |
| **Embedding Compute Path** | Local ONNX Execution Engine via CPU | Local ONNX or hardware accelerated local GPU instances |
